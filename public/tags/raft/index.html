    On Distributed Consensus
    Getting a cluster of machines to agree on anything is surprisingly hard. Not hard in the way that writing a compiler is hard — hard in the way that the problem keeps shifting under your feet the moment you think you understand it. The Core Problem Imagine three servers that need to agree on the value of a single integer. Each can send messages to the others, but messages can be delayed, duplicated, or dropped. Any server can crash at any time.
